位运算
Java中位运算是二进制补码进行运算，需要计算的值转换成二进制编码补码，得到结果后再有补码转换成原码，最后转换成十进制。
二进制的最高位是符号位，0表示正，1表示负。
正数的原码反码补码都是一样的
负数的原码进行反码补码才需要进行运算
反码：除了符号位，每位取反
补码：原码取反加一

1.^(亦或运算) ，针对二进制，相同的为0，不同的为1

2.&（与运算） 针对二进制，只要有一个为0，就为0

3.<<(向左位移) 针对二进制，转换成二进制后向左移动3位，后面用0补齐

4.>>(向右位移) 针对二进制，转换成二进制后向右移动3位，右移左移符号位不变

5.>>>(无符号右移)  无符号右移，忽略符号位，空位都以0补齐符号位跟随一起改变

应用场景

计算机中都是通过补码来进行运算的，所以二进制可以用于进行一些权限的管理，对数值的判断
为什么要使用位运算进行判断，位运算是底层二进制进行计算，运行速度上更快，一些权限管理和判断上面也更加方便
